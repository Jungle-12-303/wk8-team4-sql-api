# 요청 요약

- README의 실행 흐름을 터미널/argv 단계 없이 파일 기준 흐름만 남기도록 수정한다.

# 결정 사항

- 실행 흐름은 `main.c -> app.c -> tokenizer.c -> parser.c -> executor.c -> storage.c / schema.c`만 표시한다.
- 설명 표도 파일 기준 역할만 남긴다.

# 현재 브랜치 상태

- `simplify` 브랜치에서 작업했다.

# 완료한 작업

- `README.md`의 실행 흐름 다이어그램을 파일 기준 흐름으로 수정했다.
- 실행 흐름 표를 파일 중심 역할 설명으로 간소화했다.
