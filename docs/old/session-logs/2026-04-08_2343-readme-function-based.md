# 요청 요약

- 기존 README 문장을 참고해 다듬는 방식이 아니라, 현재 코드베이스를 기준으로 발표용 README를 새로 작성한다.
- 흐름은 함수 기준으로 설명하고, 핵심 구조체 관계도부터 실행까지 이어지도록 구성한다.
- 팀 포인트로 tokenizer/parser의 에러 메시지 처리와 회고의 협업 방식을 포함한다.

# 결정 사항

- README 구조를 `핵심 구조체 관계도 -> 함수 기준 전체 실행 흐름 -> 팀 포인트 -> 실행 예시 -> 회고` 순서로 재구성한다.
- 기존 README 문장은 유지하지 않고 새 문서로 교체한다.
- 회고에는 `main` 브랜치 `AGENTS.md`의 멀티 페르소나 리뷰 맥락과 현재 팀의 프롬프트 기반 협업 방식을 반영한다.

# 현재 브랜치 상태

- `simplify` 브랜치에서 작업했다.
- 작업 시작 시 추적되지 않은 `demo-data/` 디렉터리가 있었다.

# 완료한 작업

- tokenizer/parser의 세부 오류 메시지와 위치 정보 출력 방식을 팀 포인트 섹션으로 반영했다.
