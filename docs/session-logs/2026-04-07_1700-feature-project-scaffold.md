# 세션 로그

## 요청 요약

- 초심자 친화형 C99 SQL 처리기와 Git/GitHub 워크플로를 실제로 구현한다.

## 결정 사항

- 구현 기준은 C99다.
- 데이터는 CSV, 인덱스는 디스크 영속형 B+ 트리를 사용한다.
- 개발 흐름은 `main -> dev -> feature/*`다.
- 기능 브랜치마다 별도 세션 로그 파일을 둬 merge 충돌을 줄인다.

## 현재 브랜치 상태

- `main`에서 `dev`를 만들었다.
- `dev`에서 `feature/project-scaffold` 브랜치를 만들었다.

## 완료한 작업

- 저장소와 원격 연결 상태를 확인했다.
- 프로젝트 스캐폴딩 파일을 추가했다.
- `make`와 `make test`가 통과하는 기본 빌드 환경을 만들었다.
- 이후 코어 소스가 늘어나도 테스트 바이너리가 자동으로 링크되게
  Makefile 구조를 조정했다.
- `build` 디렉터리 선행 생성 규칙을 분리해 언제 실행해도 안정적으로
  테스트가 돌아가게 고쳤다.

## 리뷰 결과

- 테스트 링크 누락 위험을 수정했다.
- 공용 세션 로그 충돌 위험을 줄이기 위해 브랜치별 로그 파일로 바꿨다.
- README 실행 예시를 보강했다.
- 공용 헤더 변경 시 객체 파일이 다시 빌드되도록 Makefile 의존성을
  추가했다.
