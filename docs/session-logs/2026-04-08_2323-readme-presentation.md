# 요청 요약

- 프로젝트 발표자료로 활용할 수 있도록 `README.md`를 먼저 발표용 구조로 재작성한다.

# 결정 사항

- README 앞부분은 발표용 설명 중심으로 재구성한다.
- 문서 내용은 현재 구현 범위인 `INSERT`, `SELECT`, CSV 헤더 자동 생성, 스키마 기반 검증만 반영한다.
- 저장소 문서 역할도 유지하도록 실행 방법, 테스트 범위, 코드 읽기 시작점도 함께 남긴다.

# 현재 브랜치 상태

- `simplify` 브랜치에서 작업했다.
- 작업 시작 시 추적되지 않은 `demo-data/` 디렉터리가 있었다.

# 완료한 작업

- `README.md`를 발표용 흐름에 맞게 전면 재작성했다.
- 프로젝트 목적, 범위 축소 이유, 핵심 파이프라인, 실행 예시, 발표 포인트를 추가했다.
- 실제 코드 기준으로 지원 범위와 테스트 범위를 다시 맞췄다.
- `make`, `make test`, 새 임시 데이터 디렉터리 기준 `examples/demo.sql` 실행으로 문서 예시를 검증했다.

# 리뷰 결과

- 현재 코드베이스는 단일 테이블, `INSERT`, `SELECT`, CSV 저장 흐름에 집중되어 있고 README도 그 범위에 맞게 정리됐다.
- 발표용 문서로서 "왜 이 범위를 택했는지" 설명이 기존보다 명확해졌다.
- README의 예시 출력은 실제 실행 결과와 일치함을 확인했다.

- 필요하면 GUIDE와 README 사이의 중복 표현을 조정한다.
