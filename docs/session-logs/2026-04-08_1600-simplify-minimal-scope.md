# 요청 요약

- 프로젝트에서 현재 목표 범위를 벗어난 기능 흔적과 산출물을 제거한다.

# 결정 사항

- 실행 방식은 SQL 파일 실행 모드만 유지한다.
- 인덱스, `WHERE`, REPL, Primary Key 관련 과거 로그와 자산은 정리한다.
- 코어 기능은 `INSERT`, `SELECT`, CSV 헤더 자동 생성만 유지한다.

# 현재 브랜치 상태

- `simplify` 브랜치에서 작업했다.

# 완료한 작업

- `src/main.c`의 사용법과 주석에서 `--index-dir`, REPL 설명을 제거했다.
- `AGENTS.md`의 검증 예시를 현재 CLI 형식에 맞게 수정했다.
- 인덱스 디렉터리 추적 파일을 삭제했다.
- 현재 범위와 맞지 않는 과거 세션 로그를 삭제했다.
- 추적되지 않은 생성 산출물과 초과 예제 파일을 삭제했다.
- `make`, `make test`를 실행해 축소 후 회귀를 확인했다.

# 리뷰 결과

- 저장소 최상위에서 현재 코어 코드는 `INSERT`, `SELECT` 중심으로 유지되는 것을 확인했다.
- 문서와 잔여 자산이 실제 구현보다 넓은 범위를 암시하던 문제가 있었다.
