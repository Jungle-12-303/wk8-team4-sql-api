# 요청 요약

- 발표용 README를 실제 파일에 반영한다.
- 핵심 구조체 관계도에 구조체 사용 이유와 장점을 추가한다.
- 실행 흐름과 핵심 구조체 관계도 순서를 바꾸고, `main.c`를 진입점으로 명시한다.
- 실행 흐름과 팀 포인트의 예시 부분에 시각 자료를 추가한다.
- parser 단계 설명에 구조체 예시를 넣고, 협업과 회고에 작업 플로우 다이어그램을 추가한다.

# 결정 사항

- README 순서를 `시스템 구조 -> 핵심 구조체 관계도 -> 실행 흐름 -> 지원 SQL -> 팀 포인트 -> 협업과 회고`로 정리한다.
- 핵심 구조체는 요청한 표 형식을 그대로 사용한다.
- 예시 이미지는 외부 이미지 대신 mermaid 다이어그램으로 넣는다.
